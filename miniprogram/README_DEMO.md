## 🎮 完整体验流程

### 场景1：顾客点餐流程

1. **浏览菜品**
   - 打开小程序，默认显示全部菜品
   - 可以切换分类查看（热菜、凉菜、汤类等）
   - 使用搜索快速找到想吃的菜

2. **加入购物车**
   - 点击菜品右侧的"+"按钮
   - 可以多次点击增加数量
   - 购物车图标会显示商品数量

3. **登录账号**
   - 点击底部"我的"标签
   - 点击"点击登录"
   - 授权微信信息
   - 登录成功

4. **提交订单**
   - 进入"购物车"页面
   - 点击"去结算"
   - 填写联系电话（如：13800138000）
   - 添加备注（如：多加辣椒，少放盐）
   - 点击"提交订单"
   - 订单提交成功！

5. **查看订单**
   - 点击"订单"标签
   - 看到所有历史订单
   - 查看订单状态和详情

6. **评价订单**
   - 订单完成后点击"去评价"
   - 对整体服务进行评分
   - 对每道菜单独评分
   - 填写评价内容
   - 提交评价

## 🔧 修改和定制

### 修改菜品数据

编辑 `miniprogram/utils/dishData.js`：

```javascript
{
  id: '19',
  name: '新菜品名称',
  description: '菜品描述',
  price: 38.0,
  emoji: '🍜',  // 选择合适的emoji
  category: 'hot',  // 分类
  categoryName: '热菜',
  isAvailable: true,
  rating: 4.8,
  soldCount: 100
}
```

### 修改主题颜色

编辑 `miniprogram/app.wxss`：

```css
:root {
  --primary-color: #52C41A; /* 主色调 */
  --primary-dark-color: #389E0D; /* 深色 */
  --primary-light-color: #D4EDDA; /* 浅色 */
}
```

### 修改配送费

编辑 `miniprogram/pages/order/order.js`，找到：

```javascript
const deliveryFee = 5.0; // 修改这里
```

## 🐛 常见问题

### Q: 数据会丢失吗？
**A:** Demo模式数据保存在本地，清除缓存会丢失。升级到云开发后数据永久保存。

### Q: 可以修改菜品吗？
**A:** 可以！编辑 `utils/dishData.js` 文件。

### Q: 如何清除测试数据？
**A:** 开发者工具 → 清除缓存 → 清除数据缓存

### Q: 真机可以运行吗？
**A:** 可以！扫码预览即可。

### Q: 可以多人同时测试吗？
**A:** Demo模式下每人的数据是独立的。需要云开发才能多人共享数据。
