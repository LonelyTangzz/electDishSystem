# ☁️ 上传云函数详细步骤

## 📍 找到cloudfunctions文件夹

在微信开发者工具的**左侧文件树**：

1. **滚动到最顶部**
2. 找到 `cloudfunctions` 文件夹（和 `miniprogram` 同级）
3. 点击展开 `cloudfunctions`

```
electDishSystem/
├── cloudfunctions/        ← 这个文件夹
│   └── login/            ← 要上传这个
└── miniprogram/
    ├── pages/
    └── ...
```

---

## 🔧 配置云环境

**重要：必须先配置环境！**

### 方法1：右键cloudfunctions文件夹

1. **右键点击** `cloudfunctions` 文件夹
2. 选择 **"当前环境"**
3. 会弹出环境选择框
4. 选择 `cloud1-7gjt5l5i0f2bf5d4`

### 方法2：从顶部菜单配置

1. 点击顶部菜单 **"云开发"**
2. 进入云开发控制台
3. 会自动关联环境

---

## 📤 上传login云函数

**配置好环境后：**

1. 展开 `cloudfunctions` 文件夹
2. 找到 `login` 文件夹
3. **右键点击 `login` 文件夹**
4. 选择 **"上传并部署：云端安装依赖"**
5. 等待上传完成（约30秒）
6. 看到绿色✓图标表示成功

---

## ✅ 验证上传成功

### 在微信开发者工具中

`login` 文件夹旁边会出现：
- 🟢 绿色云朵图标 = 已上传
- ⚪ 灰色云朵图标 = 未上传

### 在云开发控制台中

1. 点击顶部 **"云开发"** → **"云函数"**
2. 应该能看到 `login` 函数
3. 显示部署时间和版本

## 🐛 常见问题

### Q1: 找不到cloudfunctions文件夹
**A:** 
- 确保在项目根目录查看
- 如果真的没有，关闭项目重新打开
- 确认打开的是 `electDishSystem` 文件夹

### Q2: 右键没有"上传并部署"选项
**A:**
- 先配置云环境（右键cloudfunctions文件夹 → 当前环境）
- 确保已开通云开发
- 确保project.config.json中有正确的appid

### Q3: 上传失败
**A:** 查看错误信息
- 网络问题：重试
- 权限问题：检查云开发是否开通
- 环境问题：确认环境ID正确
