# 📱 Demo模式使用说明

## 🎯 什么是Demo模式？

Demo模式是为了在**没有云开发权限**的情况下，也能完整体验小程序的所有功能。

在Demo模式下：
- ✅ 所有功能都可以正常使用
- ✅ 数据保存在**本地存储**（微信提供的 `wx.storage`）
- ✅ 不需要任何云开发配置
- ✅ 可以立即运行和演示

---

## 🚀 快速开始（30秒）

### 1. 打开项目
在微信开发者工具中打开项目

### 2. 点击编译
直接点击"编译"按钮，小程序会自动以Demo模式启动

所有功能都可以正常使用！

## 🎮 完整体验流程

### 第1步：浏览菜品
1. 打开小程序，进入"点餐"页面
2. 可以看到18道精选菜品（热菜、凉菜、汤类、主食、甜品、饮料）
3. 使用搜索或分类筛选功能

### 第2步：添加购物车
1. 点击菜品右侧的"+"按钮
2. 购物车图标会显示数量徽标
3. 可以继续添加其他菜品

### 第3步：登录
1. 点击底部"我的"标签
2. 点击"点击登录"
3. 授权获取微信用户信息
4. 登录成功！

### 第4步：提交订单
1. 返回"购物车"页面
2. 点击"去结算"
3. 填写联系电话（如：13800138000）
4. 可选填写备注（如：多加辣椒）
5. 点击"提交订单"
6. 订单提交成功！

### 第5步：查看订单
1. 点击"订单"标签
2. 可以看到刚刚提交的订单
3. 订单状态为"待确认"

### 第6步：评价订单
1. 等待订单状态变为"已完成"（Demo模式下订单立即完成）
2. 点击"去评价"按钮
3. 对整体和每道菜进行评分
4. 填写评价内容
5. 提交评价成功！

## 💾 数据存储位置

Demo模式下，所有数据都保存在微信本地存储中：

| 数据类型 | 存储Key | 说明 |
|---------|---------|------|
| 用户信息 | `demo_userInfo` | 昵称、头像等 |
| 订单列表 | `demo_orders` | 所有订单 |
| 评价列表 | `demo_reviews` | 所有评价 |
| 用户列表 | `demo_users` | 用户数据 |
| 购物车 | `cart` | 购物车商品 |

### 清除Demo数据
如果想重新开始，可以在开发者工具中：
1. 点击"清除缓存" → "清除数据缓存"
2. 重新编译小程序

## 🔄 切换到云开发模式

当您获得云开发权限后，可以轻松切换：

### 1. 修改 `app.js`
```javascript
globalData: {
  demoMode: false // 改为 false
}
```

### 2. 配置云开发环境
按照 `DATABASE_SETUP.md` 或 `QUICK_START.md` 配置云开发

### 3. 重新编译
所有功能会自动切换到云开发模式

## 📊 Demo模式 vs 云开发模式对比

| 功能 | Demo模式 | 云开发模式 |
|-----|---------|-----------|
| 数据存储 | 本地存储 | 云数据库 |
| 数据持久性 | 清除缓存会丢失 | 永久保存 |
| 多端同步 | ❌ 不支持 | ✅ 支持 |
| 数据安全 | 仅本地 | 云端加密 |
| 用户登录 | 模拟OpenID | 真实OpenID |
| 性能 | 快速 | 依赖网络 |
| 适用场景 | 演示、测试 | 正式上线 |

## 🐛 常见问题

### Q1: 为什么我的订单消失了？
**A:** Demo模式数据保存在本地，如果：
- 清除了缓存
- 删除了小程序
- 切换了微信账号

数据会丢失。这是正常现象。

### Q2: 可以多人协作测试吗？
**A:** Demo模式下每个人的数据是独立的（保存在各自设备），无法共享。需要云开发模式才能多人协作。

### Q3: Demo模式的订单状态会变化吗？
**A:** Demo模式下订单状态是静态的，不会自动变化。真实场景需要后台管理系统来更新订单状态。

### Q4: 可以添加更多菜品吗？
**A:** 可以！编辑 `miniprogram/utils/dishData.js` 文件，按照现有格式添加更多菜品。
